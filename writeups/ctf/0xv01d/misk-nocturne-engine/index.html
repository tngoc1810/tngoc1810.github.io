Misk Nocturne Engine

Reverse engineering a Windows PE64 checker that hides the real validation logic behind a fake flag, target bytes, eight transition tables and a state machine that remembers previous transitions.

Executive Summary

In this challenge, the binary exposes a fake flag through strings: 0xV01D{eight_states_are_not_enough}. However, the real checker uses a finite-state-style validation routine with eight state tables, a memory variable, a 42-byte target array and five transitions per input character.

By extracting the target bytes and state tables from the PE file, then re-implementing the validation logic in Python, the correct flag can be recovered through printable ASCII brute force.

Lab Overview

Challenge description: 

Misk Nocturne Engine
500

The machine has only eight states, but it remembers every wrong transition.

The phrase only eight states strongly hints at an automaton or state machine. The phrase remembers every wrong transition suggests that the logic is not stateless and likely carries some memory between characters.

Key Findings

1. Visible flag is bait

Running strings reveals a flag-like value: 

0xV01D{eight_states_are_not_enough}

But the binary also contains this warning: 

operator note: the visible state names are bait

This confirms that the visible string should not be trusted directly.

2. Input length must be 42 bytes

The checker compares the input length against 0x2a: 

cmp rax, 0x2a
jne fail

Since 0x2a = 42, the valid flag must be exactly 42 characters long.

3. Target array and state tables are in .rdata

target array:              0x140005080
state table pointer array: 0x1400050c0

The state table pointer array contains eight table pointers: 

0x140005800
0x140005700
0x140005600
0x140005500
0x140005400
0x140005300
0x140005200
0x140005100

Step 4 — Reconstruct checker logic

The validation logic can be represented as: 

if (strlen(input) != 42) {
    fail();
}

state = 3;
memory = 0x41;

for (i = 0; i < 42; i++) {
    x = ((i * 9 + memory) & 0xff) ^ input[i];

    for (j = 0; j < 5; j++) {
        table_id = (state + j) & 7;
        x = table[table_id][x];

        memory = (19 * state + x + j + memory) & 0xff;
        state = (5 * state + x + i + j) & 7;
    }

    out = x ^ memory;

    if (out != target[i]) {
        fail();
    }
}

success();
Key Insight The state is limited to eight values using & 7, while the memory byte is carried forward across transitions. That matches the challenge hint: the machine has eight states, but remembers transitions.

Solver Script

The solver is long, so it is hidden by default. Click the block below to expand and read the full script.

View Solver Script 
import struct

BIN = "misk_automaton.exe"
BASE = 0x140000000

data = open(BIN, "rb").read()

# Parse PE header
pe = struct.unpack_from("<I", data, 0x3c)[0]
num_sections = struct.unpack_from("<H", data, pe + 6)[0]
opt_size = struct.unpack_from("<H", data, pe + 20)[0]
sec_off = pe + 24 + opt_size

sections = []

for i in range(num_sections):
    off = sec_off + i * 40
    name = data[off:off + 8].rstrip(b"\x00").decode(errors="ignore")
    vsize, vaddr, rawsize, rawptr = struct.unpack_from("<IIII", data, off + 8)
    sections.append((name, vaddr, vsize, rawptr, rawsize))

def va_to_off(va):
    rva = va - BASE

    for name, vaddr, vsize, rawptr, rawsize in sections:
        size = max(vsize, rawsize)

        if vaddr <= rva < vaddr + size:
            return rawptr + (rva - vaddr)

    raise ValueError(f"Cannot convert VA: {hex(va)}")

TARGET_VA = 0x140005080
PTRS_VA = 0x1400050c0

target = data[va_to_off(TARGET_VA):va_to_off(TARGET_VA) + 42]

ptrs = []

for i in range(8):
    ptr = struct.unpack_from("<Q", data, va_to_off(PTRS_VA) + i * 8)[0]
    ptrs.append(ptr)

tables = []

for ptr in ptrs:
    table = data[va_to_off(ptr):va_to_off(ptr) + 256]
    tables.append(table)

def check_one_char(i, ch, state, memory):
    x = ((i * 9 + memory) & 0xff) ^ ch
    x &= 0xff

    for j in range(5):
        table_id = (state + j) & 7
        x = tables[table_id][x]

        memory = (19 * state + x + j + memory) & 0xff
        state = (5 * state + x + i + j) & 7

    out = x ^ memory
    return out & 0xff, state, memory

allowed = bytes(range(32, 127))

candidates = [(b"", 3, 0x41)]

for i, expected in enumerate(target):
    new_candidates = []

    for prefix, state, memory in candidates:
        for ch in allowed:
            out, new_state, new_memory = check_one_char(i, ch, state, memory)

            if out == expected:
                new_candidates.append((prefix + bytes([ch]), new_state, new_memory))

    candidates = new_candidates

    print(f"index={i}, candidates={len(candidates)}")

print("\nFinal candidates:")

for flag, state, memory in candidates:
    print(flag.decode(errors="replace"), "state =", state, "memory =", memory)

Lessons Learned

  • Do not trust visible strings immediately in reverse engineering challenges.
  • Fake flags are common in CTF binaries and must be verified against program logic.
  • Hints often describe the actual algorithm, not just the story.
  • When lookup tables are involved, emulation is usually better than manual table analysis.
  • Small state spaces make brute forcing practical when the checker can be reproduced accurately.
