Orbital Docking Handshake

The docking console hides its handshake phrase in a runtime-built buffer behind one more computed alignment value. Use the lightest reversing path and recover both values.

Category
Reverse Engineering
Difficulty
Medium
Architecture
ARM 64-bit (macOS/iOS)

Overview

Đây là định dạng executable của macOS/iOS kiến trúc ARM 64-bit, có cơ chế bảo vệ PIE nhưng không ảnh hưởng đến static analysis.

Static Analysis

Tiến hành decompile file bằng Ghidra, ta thấy chương trình đã gán __stack_chk_guard. Điều này tái khẳng định việc dùng phương pháp Buffer Overflow để đè biến là không khả thi.

Chương trình in ra dòng: "Hint for analysts: the shortest path is still the cleanest one."

Hỏi user nhập Docking phrase, đọc tối đa 0x40 = 64 ký tự, lưu vào acStack_58. Nếu user không nhập gì thì báo lỗi thoát. Vậy acStack_58 là buffer chứa phrase user nhập.

Tiếp theo hỏi user nhập alignment window, đọc tối đa 0x20 = 32 ký tự vào acStack_85. Vậy acStack_85 là buffer chứa alignment user nhập.

Phân tích Docking Phrase

Tiếp theo tôi chú ý đến hàm _build_expected_phrase(). Ghidra không show tham số ở entry nhưng hàm thật sự nhận param_1. Khả năng param_1 trỏ đến auStack_65, tức là buffer để hàm ghi kết quả vào.

Vòng lặp 12 lần, mỗi lần tăng 1, khả năng phrase dài 12 kí tự.

Đọc byte thứ i từ obfuscated_phrase, lưu vào bVar1. Sau đó gọi hàm mask_for truyền local_20 vào lưu vào bVar2. Tiếp theo thấy XOR 2 byte lại và lưu vào buffer:

buffer[i] = obfuscated_phrase[i] ^ mask_for(i)

Cuối cùng gán phần tử thứ 12 là 0 để kết thúc chuỗi.

Kiểm tra hàm mask_for(i): Nó trả về i * 17 + 27. Vậy là buffer[i] = obfuscated_phrase[i] ^ (i * 17 + 27).

12 bytes của obfuscated_phrase là: 7f, 43, 5e, 25, 37, 11, ef, f6, d0, cd, ab, b5
Chuyển sang số thập phân: 127, 67, 94, 37, 55, 17, 239, 246, 208, 205, 171, 181

Script tính Phrase:

obfuscated_phrase = [127, 67, 94, 37, 55, 17, 239, 246, 208, 205, 171, 181]
mask_for = [27, 44, 61, 78, 95, 112, 129, 146, 163, 180, 197, 214]
decrypted_phrase = ""

for i in range(len(obfuscated_phrase)):
    decrypted_byte = obfuscated_phrase[i] ^ mask_for[i]
    decrypted_phrase += chr(decrypted_byte)

print(decrypted_phrase)

Giờ ta đã biết phrase là: dockhandsync

Phân tích Alignment Window

Tiếp theo coi thử hàm compute__grid_offset. Nó duyệt hết chuỗi 12 kí tự của phrase. Biến sum cộng dồn với phrase[i] * (i + 3).

Kết quả return (sum % 1000) + 200.

Script tính Alignment:

obfuscated_phrase = [127, 67, 94, 37, 55, 17, 239, 246, 208, 205, 171, 181]
decrypted_phrase = ""
total_sum = 0

for i in range(len(obfuscated_phrase)):
    mask = (i * 17 + 27) & 0xFF
    char_val = obfuscated_phrase[i] ^ mask
    decrypted_phrase += chr(char_val)
    total_sum += char_val * (i + 3)

alignment_window = (total_sum % 1000) + 200
print(alignment_window)

Giờ ta đã biết alignment window là: 1108

Khôi phục Flag

Nhìn vòng lặp cuối cùng, ta biết flag dài 33 bytes. Tôi lấy 33 bytes của encoded_flag: 0xda, 0xa1, 0xb5, 0xad, 0xa4, 0xa8, 0x9b, 0xa0, 0xdf, 0x88, 0x96, 0xdf, 0x80, 0x30, 0x91, 0x55, 0x68, 0x3a, 0x7f, 0x7c, 0x1a, 0x58, 0x57, 0x75, 0x42, 0x70, 0x4c, 0x32, 0x79, 0x28, 0x6b, 0x2e, 0x1a.

Công thức logic trong vòng lặp giải mã flag là:

flag[i] = encoded_flag[i] ^ (i*5 + phrase[i%12] + alignment)

Script tính Flag:

encoded_flag = [
    0xda, 0xa1, 0xb5, 0xad, 0xa4, 0xa8, 0x9b, 0xa0,
    0xdf, 0x88, 0x96, 0xdf, 0x80, 0x30, 0x91, 0x55,
    0x68, 0x3a, 0x7f, 0x7c, 0x1a, 0x58, 0x57, 0x75,
    0x42, 0x70, 0x4c, 0x32, 0x79, 0x28, 0x6b, 0x2e,
    0x1a
]
phrase = "dockhandsync"
alignment = 1108
flag = ""

for i in range(len(encoded_flag)):
    phrase_char_val = ord(phrase[i % 12])
    mask = (i * 5 + phrase_char_val + alignment) & 0xFF
    decrypted_char = chr(encoded_flag[i] ^ mask)
    flag += decrypted_char

print(flag)

Chạy script trên ta sẽ nhận được flag hợp lệ.
