Selected CTF writeups from NovaRun team practice and TJCTF participation. This page is where I organize team-related notes, challenge links, and lessons learned from competition problem solving.

Key Insight A useful CTF writeup should show the thinking path, not just commands or the final script.

Analysis

This section can be used as the introduction page for TJCTF team activity. Later, when you finish a challenge, you can link each real writeup here.

My usual workflow is: identify the category, inspect the artifact or service, note down failed ideas, then document the actual solution path in a clean and reusable format.

Technical Notes

You can use this page as a category hub. For example:

  • Pwn: heap, format string, libc leak, safe-linking, UAF.
  • Reverse: string deobfuscation, control-flow recovery, key checking logic.
  • Web: input handling, access control, SSRF, template injection.
  • Forensics / OSINT: evidence chain, timeline reconstruction, IOC collection.

Later, each completed writeup will live in its own path, for example /writeups/ctf/tjctf/bit-leak/ or similar.

Lessons Learned

Team competitions improve more than just solving speed:

  • How to communicate ideas clearly under time pressure.
  • How to write reusable notes after solving a challenge.
  • How to convert a temporary solution into a polished public writeup.
  • How to document reasoning in a way that is useful to recruiters and teammates.
