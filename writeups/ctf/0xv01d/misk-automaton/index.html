Misk Automaton

Reverse engineering a Windows PE64 checker that hides the real validation logic behind a fake flag, target bytes, eight transition tables and a small automaton state machine.

Overview

The challenge provides a ZIP archive named 9znj0v.zip. After extracting it, the main artifact is misk_automaton.exe, a Windows executable. Since this is a 500-point reverse engineering challenge, the expected path is not to trust obvious strings but to recover the actual validation algorithm.

Initial Triage

File Type

First, I checked the binary type. This confirms that the artifact is a Windows 64-bit PE executable and that the binary is stripped to an external PDB, so the reverse process cannot rely on friendly symbol names.

Command
file misk_automaton.exe
Output
misk_automaton.exe: PE32+ executable for MS Windows 5.02 (console), x86-64 (stripped to external PDB), 11 sections

Because this is Windows x64, the calling convention matters when reading assembly: rcx is the first argument, rdx is the second argument, r8 is the third argument, and r9 is the fourth argument.

Fake Flag in Strings

I then ran strings to locate obvious messages, success strings, failure strings or embedded flags.

Command
strings -a misk_automaton.exe | grep -E '0xV01D|bait|operator|Correct|Wrong|usage'
Output
operator note: the visible state names are bait
0xV01D{eight_states_are_not_enough}

The string 0xV01D{eight_states_are_not_enough} looks like a flag, but the nearby phrase operator note: the visible state names are bait makes it highly suspicious.

Automaton Logic

The main loop processes one input byte at a time. It first mixes the byte with the current index and state1.

x = ((i * 9 + state1) & 0xff) ^ input[i]

5-Round Transition

Then the checker runs five transition rounds. The selected table depends on state2 and the round index j.

for (j = 0; j < 5; j++) {
    table_index = (state2 + j) & 7;
    x = table[table_index][x];

    state1 = (state2 * 19 + x + j + state1) & 0xff;
    state2 = (state2 * 5 + x + i + j) & 7;
}

Target Comparison

After the transition rounds, the checker compares x ^ state1 with the corresponding target byte.

if ((x ^ state1) != target[i]) {
    wrong();
}
Key InsightThe checker is stateful, but each position still compares against target[i]. If the current states are known, I can brute force the next character and carry the new state forward.

Solver

The solver extracts the target bytes and the eight transition tables from the PE file, then uses DFS/backtracking over a limited flag charset.

from pathlib import Path
import struct
from functools import lru_cache

path = "misk_automaton.exe"
data = Path(path).read_bytes()

BASE = 0x140000000

SECTIONS = [
    (".text",  0x1000, 0x2378, 0x0400),
    (".data",  0x4000, 0x0120, 0x2800),
    (".rdata", 0x5000, 0x17d0, 0x2a00),
]

def va_to_off(va):
    rva = va - BASE
    for name, sec_rva, sec_size, sec_off in SECTIONS:
        if sec_rva <= rva < sec_rva + sec_size:
            return sec_off + (rva - sec_rva)
    raise ValueError(hex(va))

TARGET_VA = 0x140005080
PTRS_VA = 0x1400050c0

target = data[va_to_off(TARGET_VA):va_to_off(TARGET_VA) + 0x2a]

ptrs = []
for i in range(8):
    ptr = struct.unpack_from("<Q", data, va_to_off(PTRS_VA) + i * 8)[0]
    ptrs.append(ptr)

tables = []
for ptr in ptrs:
    off = va_to_off(ptr)
    tables.append(data[off:off + 256])

def step(i, ch, state1, state2):
    x = ((i * 9 + state1) & 0xff) ^ ch
    for j in range(5):
        table_index = (state2 + j) & 7
        x = tables[table_index][x]
        state1 = (state2 * 19 + x + j + state1) & 0xff
        state2 = (state2 * 5 + x + i + j) & 7
    return x ^ state1, state1, state2

charset = "0xV01D{}abcdefghijklmnopqrstuvwxyzABCDEFGHIJKLMNOPQRSTUVWXYZ0123456789_"
charset = list(dict.fromkeys(map(ord, charset)))

@lru_cache(None)
def dfs(i, state1, state2):
    if i == len(target):
        return b""
    for ch in charset:
        out, new_state1, new_state2 = step(i, ch, state1, state2)
        if out == target[i]:
            tail = dfs(i + 1, new_state1, new_state2)
            if tail is not None:
                return bytes([ch]) + tail
    return None

flag = dfs(0, 0x41, 3)
print("[+] flag:", flag.decode())

Lessons Learned

What I learned
  • Never trust an obvious flag from strings in a hard reverse challenge.
  • Checking where a suspicious string is referenced is more reliable than guessing.
  • PE virtual addresses must be converted to file offsets before extracting data in Python.
  • A stateful checker can still be brute forced position-by-position if the state transition is fully modeled.

This challenge hides the real validation behind an automaton made of two states, eight transition tables, 42 target bytes and five transition rounds per character. After translating the assembly into pseudocode and simulating the logic, the true flag becomes recoverable.
