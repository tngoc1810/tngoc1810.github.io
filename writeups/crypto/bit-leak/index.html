This writeup covers the Bit Leak challenge, a classic RSA parity oracle attack. The server provides an oracle that decrypts a chosen ciphertext and returns only the least significant bit of the plaintext.

Key InsightA single leaked bit can break RSA when the oracle is repeatable and the ciphertext can be transformed using RSA multiplication.

Context

RSA has a multiplicative property: Enc(m1) * Enc(m2) mod n = Enc(m1 * m2 mod n). This lets us transform ciphertexts without knowing the private key.

Analysis

By multiplying the ciphertext by pow(2, e, n), the decrypted plaintext becomes 2m mod n. The parity of that value tells us whether m is in the lower or upper half of the current interval.

Workflow

from fractions import Fraction\nfrom Crypto.Util.number import long_to_bytes\n\nlow = Fraction(0)\nhigh = Fraction(n)\nmult = pow(2, e, n)\ncur = c\n\nfor _ in range(n.bit_length()):\n    cur = (cur * mult) % n\n    bit = oracle(cur)\n    mid = (low + high) / 2\n    if bit == 0:\n        high = mid\n    else:\n        low = mid\n\nprint(long_to_bytes(int(high)))

Technical Notes

  • Use rational intervals instead of floating point.
  • Each oracle response halves the plaintext search space.
  • The attack works because RSA encryption is malleable.

Lessons Learned

What I learned:
  • A single bit leak is dangerous in cryptographic protocols.
  • Oracle attacks often turn tiny leaks into full recovery.
  • Mathematical precision matters in exploit scripts.
