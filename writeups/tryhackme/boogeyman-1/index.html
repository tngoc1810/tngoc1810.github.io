Boogeyman 1: Phishing to DNS Exfiltration

A SOC investigation writeup covering a finance-targeted phishing email, malicious LNK execution, PowerShell C2, Seatbelt enumeration, Sticky Notes credential discovery, KeePass database theft and DNS-based exfiltration.

Investigation Goal Reconstruct how the attacker compromised Julianne Westcott's workstation, identify the payload chain, determine what sensitive data was accessed, and prove the exfiltration method using endpoint and network evidence.

Scenario Overview

Julianne Westcott, a finance employee, received a follow-up email about an unpaid invoice. The message looked business-related, but the attached archive contained a malicious Windows shortcut file. When opened, the shortcut executed an encoded PowerShell command that downloaded and ran a remote script.

The investigation started from the email artifact and pivoted into PowerShell logs and packet capture evidence. The final chain showed credential discovery through Microsoft Sticky Notes and exfiltration of a KeePass database through DNS queries.

SOC mindset: I treated each artifact as a pivot point: email headers explain initial access, LNK metadata explains execution, PowerShell logs explain endpoint impact, and PCAP traffic confirms C2 plus exfiltration.

Artifacts and Tools

  • dump.eml: phishing email sample with headers and attachment.
  • powershell.json: JSON-formatted PowerShell logs extracted from EVTX.
  • capture.pcapng: network traffic from Julianne's workstation.
  • Thunderbird, lnkparse, jq, Wireshark, tshark and CyberChef: used for evidence extraction and decoding.

1. Email Analysis: Initial Access

The phishing email claimed to be a payment collection notice and included an encrypted ZIP attachment. The archive password was written directly in the email body, a common trick to bypass automated inspection.

From: Arthur Griffin <agriffin@bpakcaging.xyz>
To: Julianne Westcott <julianne.westcott@hotmail.com>
Subject: Collection for Quick Logistics LLC - Jan 2023
Attachment: Invoice.zip

Header analysis also revealed Elastic Email infrastructure as the third-party relay service.

2. LNK Payload Analysis

After extraction, the archive contained Invoice_20230103.lnk. Using lnkparse, I found an encoded PowerShell command in the shortcut command-line field. Decoding it revealed a remote download-and-execute payload that contacted files.bpakcaging.xyz.

Why this matters: hidden encoded PowerShell plus remote script execution is a strong malicious execution indicator.

3. PowerShell Logs: C2 Setup

PowerShell logs confirmed the payload execution under QUICKLOGISTICS\j.westcott. The script configured cdn.bpakcaging.xyz:8080 as the command-and-control endpoint and used HTTP requests for tasking and output.

The two main attacker-controlled domains were files.bpakcaging.xyz for file hosting and cdn.bpakcaging.xyz for C2.

4. Enumeration: Seatbelt

The attacker downloaded and executed sb.exe, which the logs identified as Seatbelt. Seatbelt is a Windows enumeration tool commonly used after compromise to collect user, system and security configuration information.

5. Sticky Notes Discovery with sq3.exe

The attacker downloaded sq3.exe, likely a renamed SQLite command-line utility, and used it to query the victim's Microsoft Sticky Notes database: C:\Users\j.westcott\AppData\Local\Packages\Microsoft.MicrosoftStickyNotes_8wekyb3d8bbwe\LocalState\plum.sqlite.

6. KeePass Targeting and DNS Exfiltration

The attacker targeted a KeePass database named protected_data.kdbx. The PowerShell logs showed that the file was read as bytes, converted into hexadecimal, split into chunks and sent out through DNS queries using nslookup.

7. Network Evidence

PCAP analysis confirmed the same behavior. HTTP responses from the file server showed SimpleHTTP/0.6 Python/3.10.7, indicating a Python SimpleHTTP server. C2 command output was returned through HTTP POST. Exfiltration appeared as DNS traffic to 167.71.211.113 with hex chunks embedded in subdomains.

Lessons Learned

  • Password-protected archives can hide malicious payloads from automated scanning.
  • LNK files are high-risk attachments because they can run hidden PowerShell commands.
  • PowerShell logs are valuable for reconstructing attacker intent and endpoint impact.
  • PCAP evidence can validate C2 behavior and show the exfiltration channel.
  • DNS traffic should be reviewed for high-entropy or unusually long subdomains.
  • Sticky Notes and local password databases are high-value targets during endpoint compromise.
Final Assessment The attacker compromised the workstation, discovered sensitive credential material through local application data, exfiltrated a KeePass database using DNS and exposed stored financial data. Recommended response actions include isolating the endpoint, rotating credentials, blocking attacker infrastructure, reviewing DNS logs for similar patterns and improving controls for encrypted archives and LNK attachments.
