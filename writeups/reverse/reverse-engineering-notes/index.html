Ghidra / ELF / Solver

Notes about binary checkers, string obfuscation, algorithm reconstruction and solver writing.

Key InsightThe goal is not only to recover the flag, but to understand how the binary validates input.

Context

Reverse engineering practice usually starts from strings, functions, control flow and data transformations.

Analysis

I focus on turning assembly or decompiled code into a clean algorithm, then writing a solver.

Workflow

1. strings/file/checksec\n2. Open in Ghidra\n3. Find main/check function\n4. Rename variables\n5. Reconstruct algorithm\n6. Write solver

Technical Notes

  • ELF analysis
  • Ghidra decompiler
  • XOR and byte transformations

Lessons Learned

What I learned:
  • Renaming variables makes hard code readable.
  • Most checkers become simple algorithms.
  • Solvers are proof of understanding.
