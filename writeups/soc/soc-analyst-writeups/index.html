SOC-Analyst-Writeups

A collection of blue team labs, investigation notes, log analysis practice and SOC learning writeups.

Key InsightA good SOC writeup should show evidence, timeline, investigation reasoning and final conclusion.

Context

This page will collect lab reports from blue team training paths such as Cyberjutsu Blue Team and Blue Team Junior Analyst Pathway.

Analysis

For each lab, I document alert context, evidence collected, timeline, indicators, root cause and response recommendation.

Workflow

1. Collect alert context\n2. Identify affected host/user\n3. Review logs and network evidence\n4. Build timeline\n5. Extract indicators\n6. Recommend response

Technical Notes

  • Windows Event Logs
  • SIEM queries
  • Network traffic analysis

Lessons Learned

What I learned:
  • Clear documentation matters in SOC work.
  • Evidence must support every conclusion.
  • Timeline building is a core skill.
