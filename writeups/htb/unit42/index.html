HTB Sherlock: Unit42

A Sysmon-based investigation of a Windows host infected by a malicious UltraVNC-based payload. I followed the activity from download evidence to process execution, timestomping, dropped files, DNS queries, outbound connection and process termination.

Investigation goal: reconstruct how the victim downloaded and executed a malicious file, identify the cloud service used for delivery, confirm timestomping, locate dropped files and map the DNS/network activity.

Scenario Overview

This Sherlock is inspired by a backdoored UltraVNC campaign. The victim executed a suspicious installer-like file from the Downloads folder. After execution, the payload queried a dummy domain, connected outbound over HTTP, dropped files in the user's profile, changed file creation time metadata and terminated itself.

What I thought first: because the artifact is Sysmon, I should not start with a single IOC. I need to reconstruct the chain with event IDs: download/file creation, process creation, DNS, network, dropped files, timestomp and termination.

Methodology

The artifact was Microsoft-Windows-Sysmon-Operational.evtx. I opened it in Event Viewer and filtered the Sysmon log by event IDs that map naturally to malware behavior.

Event ID 1 → Process Creation Event ID 2 → File Creation Time Changed / Timestomp Event ID 3 → Network Connection Event ID 5 → Process Termination Event ID 11 → File Created Event ID 22 → DNS Query

Event ID 11 Count

Filtering Event ID 11 showed 56 file creation events. I used this count as the first answer, but more importantly it told me there were enough file creation records to identify download and dropped artifacts.

Malicious Process

I pivoted to Event ID 1 because process creation shows what actually executed. The suspicious process was:

C:\Users\CyberJunkie\Downloads\Preventivo24.02.14.exe.exe

I treated this process as suspicious for several reasons at the same time: it ran from Downloads, used a double .exe.exe extension, and had odd metadata such as OriginalFileName: Fattura 2 2024.exe, Description: Photo and vn Installer, and Company: Photo and Fax Vn. The Sysmon rule mapped it to T1204 - User Execution, which fits a user running a downloaded payload.

What I checked here: a file in Downloads is not automatically malware. The stronger signal was the combination of path, double extension, suspicious metadata, execution event and hash reputation.

The SHA256 hash was reported as malicious by VirusTotal with UltraVNC / WinVNC-based labels:

0CB44C4F8273750FA40497FCA81E850F73927E70B13C8F80CDCFEE9D1478E6F3

Cloud Drive Delivery

Next I wanted to know how the payload landed in Downloads. Filtering Event ID 22 showed Firefox querying a Dropbox content domain shortly before the executable appeared on disk:

uc2f030016253ec53f4953980a4e.dl.dropboxusercontent.com

This tied the delivery path to Dropbox. I did not need to claim that Dropbox itself was malicious; the evidence only shows the attacker abused Dropbox content hosting to deliver the file.

Defense Evasion: Timestomping

Event ID 2 showed that the malicious process changed the creation timestamp of a dropped PDF. The new timestamp was:

2024-01-14 08:10:06

Relevant fields:

TargetFilename: C:\Users\CyberJunkie\AppData\Roaming\Photo and Fax Vn\Photo and vn 1.1.2\install\F97891C\TempFolder\~.pdf
CreationUtcTime: 2024-01-14 08:10:06.029
PreviousCreationUtcTime: 2024-02-14 03:41:58.404
Why this mattered: changing file creation time is a defense-evasion clue. The file was created during the infection window, but the timestamp was backdated to January, which can make timeline review less obvious.

Dropped File: once.cmd

Filtering Event ID 11 for once.cmd revealed a dropped command script under a suspicious AppData path:

C:\Users\CyberJunkie\AppData\Roaming\Photo and Fax Vn\Photo and vn 1.1.2\install\F97891C\WindowsVolume\Games\once.cmd

The use of AppData\Roaming and misleading folders such as WindowsVolume\Games looked like staging behavior. Legitimate installers can write to AppData, but this path combined with the malicious process context made it suspicious.

DNS and Network Activity

The malicious executable queried a dummy domain, likely to check Internet connectivity:

www.example.com

Then Event ID 3 showed an outbound TCP connection to the resolved IP over port 80:

DestinationIp: 93.184.216.34
DestinationPort: 80
Protocol: tcp
How I interpreted it: www.example.com is not a normal C2 domain by itself. In this lab, it acted more like a connectivity check. The important part was that the suspicious process generated the DNS query and network connection right after execution.

Process Termination

Event ID 5 showed the malicious process terminated after completing the infection routine:

2024-02-14 03:41:58

This made the timeline tighter: execution, DNS/network activity, file drops, timestomping and termination all happened in a very short window.

IOCs and Lessons Learned

  • Malicious executable: C:\Users\CyberJunkie\Downloads\Preventivo24.02.14.exe.exe
  • SHA256: 0CB44C4F8273750FA40497FCA81E850F73927E70B13C8F80CDCFEE9D1478E6F3
  • Delivery domain: uc2f030016253ec53f4953980a4e.dl.dropboxusercontent.com
  • Dummy domain: www.example.com
  • Destination IP: 93.184.216.34
  • Dropped script: C:\Users\CyberJunkie\AppData\Roaming\Photo and Fax Vn\Photo and vn 1.1.2\install\F97891C\WindowsVolume\Games\once.cmd
Final conclusion: the victim downloaded a malicious UltraVNC-based executable from Dropbox and executed it from Downloads. The payload queried a dummy domain, connected to its resolved IP over HTTP, dropped files under AppData\Roaming, performed timestomping and then terminated.
