HawkEye Keylogger Exfiltration

Reconstructing an invoice-themed malware delivery and repeated SMTP credential exfiltration incident from a packet capture using Wireshark, CyberChef and IOC enrichment.

PlatformCyberDefenders
CategoryNetwork Forensics
DifficultyMedium
ToolsWireshark · CyberChef · VirusTotal

Executive Summary

This investigation analyzed stealer.pcap, a network trace involving an accountant who received an invoice-themed email containing a download link. Shortly after the file was downloaded, the host began sending repeated SMTP messages containing Base64-encoded password logs.

The infected workstation was identified as 10.4.10.132 / BEIJING-5CD1-PC. It downloaded a Windows executable named tkraw_Protected99.exe from proforma-invoices.com, which resolved to 217.182.138.150. The exported payload was later confirmed as malicious through hash-based enrichment.

The key finding was SMTP-based credential exfiltration to 23.229.162.69:587. The malware authenticated to the mail server using Base64-encoded credentials, then sent emails to sales.del@macwinlogistics.in. Decoding the message subject and body revealed HawkEye Keylogger - Reborn v9 password logs containing browser and Outlook credentials.

Impact: Confirmed malware download, credential theft, browser credential extraction, Outlook/POP3 credential theft, and repeated SMTP exfiltration approximately every 10 minutes.

Lab Context

The scenario begins with an accountant receiving an invoice-related email containing a download link. Suspicious traffic was observed after the link was opened. My goal was to reconstruct the incident, identify the infected host, recover the downloaded malware artifact, decode exfiltrated data, and produce a clear IOC and ATT&CK mapping.

3. HTTP Payload Delivery

I filtered for HTTP requests and found a suspicious executable download. There were no useful HTTP POST requests for exfiltration; instead, the important HTTP evidence was a GET request for a Windows executable. 

GET /proforma/tkraw_Protected99.exe HTTP/1.1
Host: proforma-invoices.com
User-Agent: Mozilla/4.0 (compatible; MSIE 7.0; Windows NT 6.1; WOW64; Trident/7.0)

The server responded with a file download:

HTTP/1.1 200 OK
Content-Type: application/x-msdownload
Content-Length: 202472
Server: LiteSpeed

Following the HTTP stream revealed MZ, PE, and the classic This program cannot be run in DOS mode string, confirming that the response body was a Windows executable rather than a document or normal invoice file.

Assessment: proforma-invoices.com was used as the invoice-themed malware delivery domain. The executable tkraw_Protected99.exe was the likely HawkEye payload.

After hashing, I treated the file as malware and renamed the extension to reduce the risk of accidental execution during analysis. 

mv tkraw_Protected99.exe tkraw_Protected99.exe.malware

I then searched the SHA256 hash in VirusTotal. The file was flagged by 57/71 security vendors and labeled with categories such as trojan, dropper, autoit, and hawkeye. This enriched the PCAP evidence and confirmed that the exported HTTP object was malicious.

Lessons Learned

This lab reinforced why an analyst should begin with capture metadata, protocol hierarchy, and endpoint statistics before diving into streams. That triage quickly separated high-volume HTTP payload delivery from lower-volume but more critical SMTP exfiltration.

The most important lesson was that Base64 is not encryption. In this case, the malware used Base64 for SMTP authentication fields, the email subject, and the message body. A simple CyberChef decode exposed the malware family, host identity, public IP address, and stolen credentials.

Finally, repeated SMTP sessions were more valuable than a single decoded credential. They showed behavior: HawkEye was configured to periodically send collected password logs, which is exactly the kind of pattern a SOC analyst should capture in a timeline, detection idea, and ATT&CK mapping.
