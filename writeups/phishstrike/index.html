PhishStrikeLab

Email header and threat-intelligence investigation of a purchase-receipt phishing email that delivered malware through a direct executable download, then led to CoinMiner, BitRAT and AsyncRAT infrastructure.

PlatformCyberDefenders
CategoryThreat Intel
TacticsInitial Access · Execution
ToolsEmail Header Analyzer · URLHaus · VirusTotal · Joe Sandbox · VMRay · CyberChef

Executive Summary

The alert involved a phishing email targeting faculty members at an educational institution. The message impersonated a trusted sender and used a high-value purchase receipt lure to pressure the recipient into opening an invoice. Header analysis showed weak authentication: SPF softfail, DKIM fail, and DMARC none. The message body revealed the real objective: a direct download link to install.exe hosted on 107.175.247.199.

Threat intelligence confirmed that this URL was not a legitimate invoice. URLHaus listed the same endpoint as a malware download URL with historical payloads tagged as CoinMiner, AsyncRAT, and BitRAT. Sandbox evidence showed staged HTTP requests, encoded PowerShell sleep, registry startup persistence, a BitRAT C2 domain, and AsyncRAT communication through Telegram Bot API.

Final assessment: PhishStrike is a phishing-to-malware-delivery case. The sender domain itself may look clean, but email authentication, body content, URL reputation, and sandbox behavior prove the email was malicious.

1. Email Header Analysis

The first phase was to inspect the visible sender, return path, recipient pattern, message ID, and authentication results. The sender appeared as ERIKA JOHANA LOPEZ VALENTE <erikajohana.lopez@uptc.edu.co>. The recipient field used undisclosed-recipients, which is common in bulk phishing campaigns.

The most important finding came from Authentication-Results. The message showed SPF softfail and DKIM fail. The sender IP associated with that authentication result was 18.208.22.104.

spf=softfail (sender IP is 18.208.22.104)
dkim=fail
smtp.mailfrom=uptc.edu.co
header.d=uptc.edu.co
dmarc=none

2. Malicious URL in the Email Body

The body claimed that an invoice document was available for confirmation, but the embedded link pointed directly to a Windows executable hosted on a raw IP address:

http://107.175.247.199/loader/install.exe

This is a high-confidence phishing indicator because invoices are normally PDF or document attachments, not a raw-IP-hosted .exe. The path /loader/install.exe also suggests malware staging.

3. Reputation and Threat Intelligence

VirusTotal showed that uptc.edu.co was not broadly flagged as malicious. The sender IP 18.208.22.104 also appeared clean. However, the payload hosting IP 107.175.247.199 was flagged as malicious and became the primary infrastructure IOC.

URLHaus confirmed that hxxp://107.175.247.199/loader/install.exe was a malware download URL. The URL was offline during later analysis, but historical intelligence still proves it was used for malware distribution.

4. CoinMiner Behavior

The cryptocurrency mining payload was associated with CoinMiner. This answers the malware-family question related to abusing system resources for cryptocurrency mining.

The sandbox process tree showed a PowerShell encoded command. Decoding the base64 string in CyberChef revealed Start-Sleep -Seconds 50.

Network artifacts showed the malware requesting staged resources from ripley.studio and 107.175.247.199.

5. BitRAT Analysis and Persistence

The BitRAT payload hash was bf7628695c2df7a3020034a065397592a1f8850e59f9a448b555bc1c8c639539. VirusTotal and Joe Sandbox both classified it as malicious. Its behavior included dropped files, network IDS alerts, encoded PowerShell, .NET unpacking indicators, and process injection.

The strongest persistence evidence came from VMRay. The sample added a dropped executable to Windows startup via registry:

C:\Users\RDhJOCNFevzX\AppData\Roaming\Ozndcoodb\Vwvix.exe

Further community intelligence identified the BitRAT command-and-control domain as gh9st.mywire.org.

6. AsyncRAT and Telegram Bot API

The AsyncRAT payload was associated with SHA256 5ca468704e7ccb8e1b37c0f7595c54df4e2f4035345b6e442e8bd4e11c58f791. Its network behavior overlapped with the same staging infrastructure and also revealed Telegram Bot API usage.
